## ファイル管理ルール

### フォルダ構成

```
CheerRING/
├── INDEX.md                  ← 全ファイル目次（常に最新に保つこと）
├── Document/
│   ├── 01_事業計画/
│   ├── 02_提案資料/
│   ├── 03_市場調査/
│   ├── 04_実装計画/
│   ├── 05_図解・画像/
│   └── 06_作業メモ/
└── （Webサイト関連ファイル: index.html, manifest.json 等）
```

### ファイル命名規則

- **連番形式**: `[NNN]_[ファイル名]`（フォルダ内で3桁連番）
- **04_実装計画のみ**: `05xxx_` 体系（CheerLink開発連番）を維持
- **作成者タグ（fumi のみ付与）**: 小林さん作成ファイルは `[NNN]_fumi_[ファイル名]`
  - 判定方法：ファイルのタイトルまたは本文に「新保さんへ」が含まれていれば `fumi` タグをつける
  - 新保さん作成ファイルには作成者タグ不要

### 作成者凡例
| コード | 人物 |
|---|---|
| `mnb` | 新保 Manabu |
| `fumi` | 小林 Fumi（CheerLink ビジネスオーナー） |
| `AI` | AI調査生成（deep-researchレポート等） |

## INDEX.md 更新ルール

### いつ更新するか
**Document/ 以下に新しいファイルを追加・移動・削除したとき、必ずその場で INDEX.md を更新すること。**

対象アクション:
- 新規ファイル作成
- ファイル移動・フォルダ整理
- ファイル削除（行を削除）
- ファイル内容の大幅改訂（ステータス変更、概要更新）

### INDEX.md の更新手順
1. ファイルの更新日を `stat -f "%Sm" -t "%Y-%m-%d" [ファイルパス]` で取得
2. 該当カテゴリのテーブルに行を追加（または既存行を更新）
3. ファイル上部の `最終更新` 日付を今日の日付に変更

### INDEX.md のカラム定義

**計画書・提案資料・調査・実装計画（01〜04, 06フォルダ）:**

| カラム | 内容 |
|---|---|
| No | フォルダ内連番（ファイル名プレフィックスと一致） |
| ファイル名 | 相対パスのMarkdownリンク |
| 種別 | MD / PPTX / PDF / PNG |
| 更新日 | ファイル最終更新日（YYYY-MM-DD） |
| 作成者 | mnb / fumi / AI |
| ステータス | `確定`（外部共有可）/ `ドラフト`（作業中）/ `参考`（内部記録用） |
| 用途 | 外部提案 / 社内 / 調査 / 開発 / 法務 |
| 概要 | ファイルの内容を1〜2行で説明 |

**05_図解・画像:**
No・ステータス・用途カラムは省略可。ファイル名・更新日・作成者・概要のみ記載。

## その他の運用メモ

- `コピーライト.md` と `コピーライト_修正版.md` が共存している場合、`修正版` が最新。旧版の削除を検討すること。
- `ロレス向け...` はファイル名の「ロ」は「プロ」の誤字。内容修正が発生した際にファイル名も合わせて修正すること。
- PDFとMDが同番号でセットになっているファイル（03_市場調査の001〜003）は、一方を削除した場合もう一方も確認すること。

## AI役割定義（Cowork / Codex 二役運用）

### 役割分担

| エージェント | 役割 | 担当ドメイン |
|------------|------|-----------|
| **Claude Cowork** | PM兼文書担当 | 戦略文書・MD起草・論点整理・タスク定義・レビュー・NOW.md管理 |
| **Codex** | 実行担当 | Excel/PPTX生成・スクリプト実行・コード修正・数値モデル管理 |

### データ所有権（正本ルール）

どちらかが同じデータを扱う場合、必ず以下の正本を参照する。独自計算・独自推測で数値を作らない。

| データ種別 | 正本ファイル | 管理者 |
|----------|-----------|------|
| 財務数値・KPI（LTV/CAC/ARR等） | `Document/07_VC資料/003_財務モデル_UnitEconomics_2026_v2.xlsx` | Codex |
| 事業戦略・市場論点・文書表現 | 各MDファイル（最新版） | Cowork |
| タスクの現在状態 | `NOW.md`（プロジェクトルート） | Cowork（PMとして管理） |
| ファイル目録 | `INDEX.md` | 両者（追加・変更時に必ず更新） |

### 跨ぎ作業プロトコル

**Codexが数値モデルを更新したとき:**
1. NOW.mdの「完了」欄に主要KPIを3行以内で書く
2. 変更したファイル名とバージョンを記録する

**CoworkがMDを作成・更新したとき:**
1. NOW.mdの「完了」欄にファイル名と変更内容を書く
2. 財務数値を含む場合は正本ファイルから転記したと明記する

**タスクを引き渡すときの必須3点セット:**
```
タスク: [タスク名]
担当: Cowork / Codex
インプット: [参照すべきファイルとシート名]
アウトプット: [作成・更新するファイル名]
正本: [数値や事実の根拠ファイル。ずれがあればこちらを優先]
```

### 禁止事項

| 禁止 | 理由 |
|-----|------|
| 財務数値をMD内で独自計算する | 正本との不整合が起きる（実際に起きた） |
| 正本ファイルを確認せずに数値を引用する | 古いバージョンの数値が残存するリスク |
| NOW.mdを確認せずにタスクを開始する | 重複作業・競合が起きる |
| INDEX.mdを更新せずにファイルを追加する | ファイル地図が崩れる |

## セッション開始時のチェックリスト

新しいセッションを開始するAIは、作業前に以下を読むこと。

1. `CLAUDE.md`（本ファイル）— ルールと役割の確認
2. `NOW.md` — 現在の状態とアクティブタスクの確認
3. `INDEX.md` — ファイル構成の確認
4. 担当タスクに関連する引継書（`Document/06_作業メモ/007_` または `008_`）
